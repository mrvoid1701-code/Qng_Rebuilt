## Minimal ontological claim

QNG is not just:

- GR on a graph
- QM on a graph
- or a bridge written in graph language

QNG claims an additional native layer:

1. a relational substrate exists
2. local state evolves by a native update rule
3. the update rule is history-sensitive
4. effective fields and geometry emerge from coarse-grained substrate behavior

This is the minimum content that makes QNG its own theory.

## Ontological levels

The rebuild distinguishes four levels:

1. `substrate ontology`
   Native relational structure and native local state.
2. `native dynamics`
   Update law on that substrate.
3. `effective field layer`
   Coarse-grained variables used to summarize native behavior.
4. `effective geometry layer`
   Geometric objects inferred from effective fields or distances.

Only levels 1 and 2 are truly ontological.  
Levels 3 and 4 are derived.

## Current best candidate for the ontic layer

The best candidate inherited from the legacy workspace is:

- dynamic graph `G(t)`
- local node state `N_i(t)`
- local update operator `U`

This is cleaner than directly declaring:

- `Sigma`
- `tau`
- effective metric

to be primitive.

## Current best candidate for native dynamics

The strongest stable native statement is:

`N_i(t+1) = U(N_i(t), neighborhood history, local noise or update inputs)`

The irreducible content is:

- locality
- discreteness
- history dependence

Everything stronger than this should currently be treated as a candidate specialization.

## Immediate derived layer

The old project strongly suggests that a stability-like field exists at the effective level.

But the rebuild should phrase that as:

- some effective field `X` summarizes substrate coherence or compatibility

rather than immediately hard-locking:

- `Sigma` as primitive ontology

This is a critical correction.

## Geometry claim at this stage

The clean ontological backbone allows only the following geometry statement:

- geometry is emergent from the effective layer

It does not yet commit to one unique generating field or one unique estimator.

## What this backbone already rules out

This backbone rejects the following shortcuts:

1. treating phenomenological `tau` as a primitive object
2. treating a successful metric estimator as ontology
3. treating `chi = m/c` as foundational before separate support exists
4. treating observational fit success as proof of the native substrate

## Working conclusion

The rebuild's current best ontological compression is:

`relational substrate + local history-sensitive update + derived effective field layer + derived emergent geometry`

That is the cleanest statement of QNG we currently have.
