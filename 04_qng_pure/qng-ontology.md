# QNG Ontology

Type: `axiom`
ID: `AX-QNG-001`
Status: `draft`
Author: `C.D Gabriel`

## Objective

State the primitive ontology of QNG only after GR and QM foundations are separated.

## Required distinction

At the QNG layer, every object must be marked as one of:

- `primitive`
- `derived`
- `effective`
- `phenomenological`

This is mandatory.

## Candidate primitive ontology

Current candidate primitives for the rebuild are:

- discrete substrate element (`node` or equivalent)
- adjacency or relational linkage
- local internal state
- update operator
- memory-bearing structure

These are candidates only. They are not locked yet.

## Candidate non-primitives

The following should be treated as derived unless forced otherwise:

- emergent metric
- effective curvature
- effective propagators
- observational fit parameters
- mission-specific amplitudes

## Hard rule

No observational fit parameter should be declared primitive here unless it is required by the ontology itself.

## Build pressure

This file exists to force the rebuild to answer:

- what QNG adds beyond GR and QM
- what QNG borrows from the bridge
- what QNG predicts internally before data enters the story

## Open work

- determine which variables are truly primitive
- separate ontological primitives from effective phenomenology parameters
- decide what is native to QNG versus inherited from bridge language
